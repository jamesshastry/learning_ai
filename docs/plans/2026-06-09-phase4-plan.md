## Phase 4A — Bug Fixes (Do First, ~2 hours)

These are real bugs found in the adversarial review that can cause incorrect behavior.

### 4A.1 — Fix model mismatch on --provider override
**Bug:** `learn ask --provider gemini` when Claude is default sends `model: "claude-sonnet-4-6"` to Gemini API → crash.
**Files:** `src/cli/ask.ts`, `src/cli/process.ts`, `src/cli/ingest.ts`, `src/utils/config.ts`
**Fix:** When `--provider` overrides config, also resolve the model name to `defaultModel(effectiveProvider)`. Pass effective model through to analyze functions.

### 4A.2 — Make writeYaml atomic
**Bug:** Ctrl+C during `writeYaml` can produce truncated/corrupt `course.yaml`.
**File:** `src/utils/files.ts`
**Fix:** Write to `{path}.tmp` then `renameSync` to final path.

### 4A.3 — Fix lecture ID prefix collision at 100+ lectures
**Bug:** `findExistingFile` searching for ID `10` also matches `100-*` directories.
**File:** `src/cli/ingest.ts`
**Fix:** Change `d.startsWith(paddedId + '-')` to exact-match: `d.split('-')[0] === paddedId`. Or use `padStart(3, '0')` consistently.

### 4A.4 — Handle orphaned directories on title change
**Bug:** When transcription returns a different title than playlist scraper, old directory stays, new one is created.
**File:** `src/cli/ingest.ts`
**Fix:** After computing new `lectureDir`, scan for any existing dir with same ID prefix but different slug. If found, rename it to the new name.

### 4A.5 — Add timeout to Claude API calls
**Bug:** Claude `messages.create()` calls have no timeout — can hang forever.
**File:** `src/pipeline/analyze.ts`
**Fix:** Add `AbortSignal.timeout(120000)` to both API calls. Add retry with backoff matching the Gemini module.

### 4A.6 — Escape `</script>` in D3.js visualization
**Bug:** Concept name containing `</script>` breaks the HTML page.
**File:** `src/graph/builder.ts`
**Fix:** `JSON.stringify(graphData).replace(/</g, '\\u003c')` in the HTML template.

### 4A.7 — Add SIGINT handler for clean shutdown
**File:** `src/cli/ingest.ts`
**Fix:** Register `process.on('SIGINT', ...)` that writes current state and breaks the loop cleanly.

### 4A.8 — Validate concepts.yaml before loading in graph builder
**Bug:** Malformed `concepts.yaml` causes confusing runtime crash.
**File:** `src/graph/merge.ts`
**Fix:** Guard with `Array.isArray(data.concepts)` and skip concepts missing `name`/`definition`.

## Phase 4B — Learning Workflow Features (Next Phase, ~1-2 sessions)

### 4B.1 — Personal progress tracking
Add learner state per lecture (separate from pipeline state).

**Data model change** — add to each lecture in `course.yaml`:
```yaml
  - id: '01'
    title: '...'
    status: completed           # pipeline status (unchanged)
    watched: true               # learner has watched
    confidence: medium          # none | low | medium | high
    watched_date: '2026-06-10'
    revisit: false              # flagged for re-review
    personal_notes: 'Jensen makes a great point about...'
```

**New commands:**
```bash
learn watch cs153 3                         # Mark as watched
learn watch cs153 3 --confidence low        # Mark with confidence
learn watch cs153 3 --revisit               # Flag for review
learn watch cs153 3 --note "Key insight..." # Add personal note
learn review                                # Show all low-confidence + flagged lectures
learn review --course cs153                 # Scoped to one course
learn progress                              # Dashboard: watched/total per course
```

### 4B.2 — External transcript import
Support non-YouTube content by accepting pre-existing transcript files.

```bash
# Import a local SRT/VTT/TXT transcript
learn import ./lecture-transcript.srt --course cs229 --title "Gradient Descent" --id 05

# Import a plain text file (already in timestamp format)
learn import ./my-notes.txt --course cs229 --title "Backpropagation" --id 06

# Process from transcript (no YouTube URL needed)
learn process --from-file ./transcript.srt --course cs229
```

**Pipeline change:** Insert a new entry point before the normalize stage — load from file instead of transcription service. All downstream stages (normalize → analyze → write) are unchanged.

### 4B.3 — Personal annotations layer
Preserve learner's own notes across reprocessing.

**Approach:** Separate `annotations.md` file per lecture (never overwritten by pipeline):
```
courses/cs153/lectures/01-.../
  transcript.txt      # pipeline-generated
  notes.md            # pipeline-generated (can be reprocessed)
  concepts.yaml       # pipeline-generated
  annotations.md      # personal (never touched by pipeline)
```

```bash
learn annotate cs153 3                  # Open annotations.md in $EDITOR
learn notes cs153 3 --with-annotations  # View notes + annotations together
```

### 4B.4 — Cross-course topic synthesis (the killer feature)
Generate a synthesis document that collects everything across all courses about a topic.

```bash
learn synthesize "transformers"
# → Searches all concept files and notes for "transformers"
# → Sends relevant excerpts to Claude/Gemini
# → Produces courses/synthesis/transformers.md with:
#    - Which courses cover this, and how deeply
#    - Key perspectives from each source
#    - Areas of agreement/disagreement
#    - Recommended reading order
#    - Combined concept map for this topic

learn synthesize "scaling laws" --output synthesis/scaling-laws.md
learn compare cs153/01 cs153/04  # Compare two specific lectures
```

### 4B.5 — Lecture range/subset selection for ingest
```bash
learn ingest cs153 --lectures 3-7       # Process lectures 3 through 7
learn ingest cs153 --lectures 3,5,8     # Process specific lectures
learn ingest cs153 --skip 7             # Skip a problematic lecture
```

### 4B.6 — Batch progress indicator
Show overall progress during `learn ingest`:
```
Processing 9 lectures in Frontier Systems (gemini)
[3/9] Lecture 05: Scott Nolan — Energy Bottlenecks
  [2/6] Transcribing...    ██████████████░░░░░░ 70%
  Overall: 3/9 lectures | ~12 min remaining
```

## Phase 4C — Export & Active Recall (Future, ~1-2 sessions)

### 4C.1 — Export to multiple formats
```bash
learn export cs153 --format pdf          # Full course study guide as PDF
learn export cs153 --format anki         # Concepts → Anki flashcards
learn export --topic "RLHF" --format md  # Cross-course topic guide
learn export cs153 3 --format md         # Single lecture clean export
```

**Anki export format:** concept name → front, definition → back, tags from concept tags, source lectures in the "Extra" field.

### 4C.2 — Quiz generation
```bash
learn quiz cs153 3                       # Generate quiz from one lecture
learn quiz cs153                         # Quiz across the whole course
learn quiz --topic "attention"           # Quiz from a specific topic
learn quiz --review                      # Quiz from low-confidence lectures
```

Uses Claude/Gemini to generate questions from notes + concepts. Multiple choice and short answer. Tracks scores per concept to identify weak areas.

### 4C.3 — "What's next?" recommendation engine
```bash
learn next                               # Suggest next lecture to watch
learn next --focus "transformers"         # Next lecture in a topic area
learn plan --hours 3                     # Build a 3-hour study plan
```

Factors: unwatched lectures, prerequisite ordering, topic balance across courses, time since last lecture in each course, low-confidence areas that need review.

### 4C.4 — Incremental graph updates
```bash
learn graph update                       # Add only new lectures since last build
learn graph update cs153                 # Update from one course only
```

Avoid full O(n²) rebuild each time — only re-merge concepts from newly processed lectures.

### 4C.5 — Add reading/resource linking
```bash
learn add-reading cs153 3 "https://arxiv.org/abs/1706.03762" --title "Attention Is All You Need"
learn add-reading cs153 3 ./slides.pdf --type slides
```

Creates `courses/cs153/lectures/03-.../readings/` directory with metadata linking supplementary materials to lectures.

## Architecture Principles for Next Phase

1. **Learner state is separate from pipeline state.** `status: completed` means the tool finished processing. `watched: true, confidence: high` means the learner learned it. Never conflate these.

2. **Personal data survives reprocessing.** Annotations, watch status, confidence, bookmarks — anything the learner creates must survive `--force` reprocessing. Use separate files or protected YAML fields.

3. **Cross-course is the differentiator.** Single-course note-taking tools exist (Notion, Obsidian). What makes this tool unique is operating *across* 10-15 courses simultaneously. Every new feature should ask: "does this work across courses?"

4. **Cost-aware by default.** Synthesis, comparison, and quiz generation all make LLM calls. Default to the cheapest adequate model (Haiku/Flash) with a `--quality high` flag for important outputs.
